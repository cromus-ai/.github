**[ETHOS.md Specification](https://github.com/cromus-ai/ethos-md-spec)** 
— An open standard for behavioral governance specs that travel with 
AI workflows. Auditable behavior, not just capable behavior.

**[MEMORY.md Specification](https://github.com/cromus-ai/memory-md-spec)** 
— An open standard for portable AI agent memory — defining what an 
agent retains across sessions, how it compresses context when limits 
are reached, and how memory transfers between agents. Works at the 
model, infrastructure, and application level universally.

## The Core Idea

Every organization has institutional knowledge — SOPs, runbooks, 
process documents — sitting in folders, never touched by AI because 
nobody compiled them into a format AI systems can actually use.

SKILL.md is that format.

A SKILL.md file captures what a skill does, how it does it, what it 
costs, what governs it, and where it runs — in plain text, readable 
by a developer, a compliance officer, and any AI framework 
simultaneously.

ETHOS.md pairs with every SKILL.md — behavioral governance baked in, 
not bolted on.

MEMORY.md completes the stack — portable, agent-readable memory that 
travels with the workflow, not with the platform.

**Intelligence precedes execution. Always.**

## Croms™

One Crom equals one unit of preventable AI workflow waste.

Croms™ is Cromus's proprietary metric for quantifying waste before 
it scales — scored across four dimensions:

- Cost Waste (35%)
- Latency Overhead (25%)
- Failure Risk (25%)
- Structural Gaps (15%)

Severity bands: Low (<100) · Moderate (100–299) · 
High (300–599) · Critical (600+)

## The Open Agent Specification Stack

Three files. Everything an AI agent needs to know what to do, 
how to behave, and what to remember — portable, auditable, 
and platform-agnostic.

| Spec | Defines | Repo |
|------|---------|------|
| SKILL.md | What the agent does | [skill-md-spec](https://github.com/cromus-ai/skill-md-spec) |
